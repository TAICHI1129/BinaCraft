# BinaCraft

**概要**
BinaCraft は、2Dグリッド上で動作する軽量なバイナリ・サンドボックスおよび論理回路プロトタイピングツールです。
テキストベースのコマンド（Abstrang形式）を使用して、電子回路や論理ゲートのシミュレーションを簡単に行うことができます。

---

## 主な特徴

* **2D セクション・システム**
  複数のグリッド（基板）をプロジェクト内で管理可能。

* **組み込みブロック**
  電源、ワイヤー、スイッチ、AND/OR/NOT ゲートを標準搭載。

* **カスタム・マクロ**
  Python の論理式形式で、独自の論理ゲート（XOR、NANDなど）を定義・再利用可能。

* **Abstrang スクリプト**
  シンプルなコマンドラインインターフェースによる回路構築。

* **永続化**
  回路構成を JSON 形式で保存・読み込み可能。

---

## インストールと実行

* **必要環境**: Python 3.7 以上
* **追加ライブラリ**: 不要

### 実行方法

1. **REPLモードの起動**
   対話形式で回路を構築

   ```bash
   python binacraft.py --repl
   ```

2. **デモの実行**
   組み込みのデモ回路を表示

   ```bash
   python binacraft.py --demo
   ```

3. **スクリプトの実行**
   ファイルに記述されたコマンドを一括実行

   ```bash
   python binacraft.py --script my_circuit.txt
   ```

## ブロック仕様

各セルは、隣接するセルの前のステップの状態（State）を読み取って動作します。

| 記号 (ON/OFF) | 種類 (kind) | 動作説明                          |
| ----------- | --------- | ----------------------------- |
| P           | Power     | 常に 1 を出力                      |
| # / :       | Wire      | 周囲（上下左右）に 1 があれば伝播            |
| T / t       | Switch    | 手動で切り替え可能な入力                  |
| A / a       | AND       | 北(N) と 東(E) が両方 1 なら出力        |
| O / o       | OR        | 上下左右（N, E, S, W）のいずれかが 1 なら出力 |
| N / n       | NOT       | 北(N) の入力を反転                   |
| M / m       | Macro     | define で定義された論理式に従い動作         |

## Abstrang コマンドリファレンス

| コマンド    | 使用例                                       | 説明                 |
| ------- | ----------------------------------------- | ------------------ |
| project | `project MyCPU`                           | プロジェクトを新規作成        |
| section | `section main 16 8`                       | 幅16、高さ8の基板を作成      |
| define  | `define XOR = (N or E) and not (N and E)` | 独自の論理式をマクロとして定義    |
| place   | `place main 2 3 wire`                     | 座標(x, y)に指定ブロックを配置 |
| toggle  | `toggle main 2 3`                         | 指定座標のスイッチを反転       |
| tick    | `tick main 5`                             | シミュレーションを5ステップ進める  |
| show    | `show main`                               | 信号状態を記号で表示         |
| signals | `signals main`                            | 信号状態を 0/1 で表示      |
| save    | `save backup.json`                        | 現在の状態をファイルに保存      |

---

## カスタム・マクロでの変数利用

`define` コマンドでは以下の予約語を変数として使用可能：

* N, E, S, W : 北・東・南・西の隣接セルの状態
* A, B, C, D : 上記と同じ（エイリアス）
* ANY : いずれかが 1 であれば真
* ALL : 全てが 1 であれば真

バイナリ（バイナリセル）を直接操作・配置して回路を組む際の実践的なテクニックと、シミュレーションの挙動について解説します。

このコードのシミュレーションは**同期更新（Synchronous Update）**を採用しており、各ティックで「全セルの前の状態」を参照して「次の状態」を一斉に計算します。

# バイナリで作ろう！

## 1. 基本的な回路構成（レイアウト）

バイナリ操作でロジックを組む際、標準ゲート（AND/OR/NOT）の入力方向には決まりがあります。

### 信号の伝播とゲートの向き
* **AND (`A`)**: 北(`N`)と東(`E`)の2方向から入力を受け取ります。西や南に電源を置いても反応しません。
* **NOT (`N`)**: 北(`N`)からの入力のみを反転します。
* **OR (`O`)**: 上下左右どこから入力しても反応する「万能接続」として使えます。



### 配置のコツ
```text
  P (電源)
  : (ワイヤー)
  A (ANDゲート) <-- 東(E)からも入力が必要
```
上記のように、ゲートの「北」と「東」にワイヤーや電源が隣接するように配置するのが基本です。

---

## 2. Abstrangでのバイナリ操作コマンド

特定の座標をピンポイントで操作する際に多用するコマンドです。

* **配置 (`place`)**
    `place セクション名 X Y 種類 [状態/マクロ名]`
    * 例: `place main 5 2 switch on` （座標5,2にON状態のスイッチを置く）
* **反転 (`toggle`)**
    `toggle セクション名 X Y`
    * 設置済みのスイッチの状態を 0→1 または 1→0 へ切り替えます。
* **一括確認 (`signals`)**
    `signals セクション名`
    * `show` コマンドが `A/a` などの記号表示なのに対し、`signals` は `0/1` の生データを表示するため、デバッグに最適です。

---

## 3. バイナリ操作による「遅延」の理解

このシステムでは、信号がワイヤー1マスを通るたびに **1ティックの遅延** が発生します。

> **重要:**
> `tick` コマンドを実行しない限り、画面上の信号（`show`の結果）は更新されません。

### 信号を遠くに送る例
1. `place main 0 0 power`
2. `place main 1 0 wire`
3. `place main 2 0 wire`
4. `show main` → この時点ではまだ `2,0` は `0` のまま。
5. `tick main 2` → 2ステップ進めると、信号が `2,0` まで到達します。

---

## 4. 応用：フィードバック回路（クロック）

バイナリセルをループさせることで、自動的に点滅するクロック回路を作れます。

```text
# クロック回路の例 (3x1)
place main 0 0 wire
place main 1 0 not  # 0,0(北)がないので反転して1を出力
place main 2 0 wire
```
※厳密には、このコードの `NOT` は「真北」のみを見るため、横一列の配置でループを作るには `macro` で「西の入力を反転する」などの定義を自作するのがスムーズです。
